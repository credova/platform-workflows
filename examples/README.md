# Examples

Starter workflows for common repo types. Each directory contains the workflow files a repo needs in `.github/workflows/`.

| Directory                         | Description                                                     |
| --------------------------------- | --------------------------------------------------------------- |
| [node-service](node-service/)     | Node.js service with container build and Cloud Run deploy       |
| [python-service](python-service/) | Python service with container build and Cloud Run deploy        |
| [ruby-service](ruby-service/)     | Ruby service with container build and Cloud Run deploy          |
| [dotnet-service](dotnet-service/) | .NET service with container build and Cloud Run deploy          |
| [kotlin-service](kotlin-service/) | Kotlin/Gradle service with container build and Cloud Run deploy |
| [go-cli](go-cli/)                 | Go CLI tool with GoReleaser (no container, no deploy)           |
| [node-sdk](node-sdk/)             | Node.js SDK/library with edge releases and npm publish          |
| [php-plugin](php-plugin/)         | PHP plugin (WordPress/WooCommerce)                              |
| [dependabot](dependabot/)         | Auto-merge Dependabot PRs                                       |
